## Kryptering av hårddiskar och filpartitionering

LUKS (Linux Unified Key Setup) följer tydligen med många moderna distros.

- Install cryptsetup-luks. (You can issue <code>apt install cryptsetup</code>, <code>yum install cryptsetup-luks</code> or <code>dnf install cryptsetup-luks</code> for Ubuntu/Debian, RHEL/Cent OS, and Fedora, respectively.)
- Confirm the partition name using <code>fdisk -l</code>, <code>lsblk</code> or <code>blkid</code>. (Create a partition using </code>fdisk</code> if necessary.)
- Set up the partition for LUKS encryption: <code>cryptsetup -y -v luksFormat /dev/sdb1</code>. (Replace <code>/dev/sdb1</code> with the partition name you want to encrypt.)
- Create a mapping to access the partition: <code>cryptsetup luksOpen /dev/sdb1 EDCdrive</code>.
- Confirm mapping details: <code>ls -l /dev/mapper/EDCdrive</code> and <code>cryptsetup -v status EDCdrive</code>.
- Overwrite existing data with zero: <code>dd if=/dev/zero of=/dev/mapper/EDCdrive</code>.
- Format the partition: <code>mkfs.ext4 /dev/mapper/EDCdrive -L "Strategos USB"</code>.
- Mount it and start using it like a usual partition: <code>mount /dev/mapper/EDCdrive /media/secure-USB</code>.

## Audit and Log Config

Bara lite kort om tail och grep, detta har jag koll på.

- <code>/var/log/messages</code> - a general log for Linux systems
- <code>/var/log/auth.log</code> - a log file that lists all authentication attempts (Debian-based systems)
- <code>/var/log/secure</code> - a log file that lists all authentication attempts (Red Hat and Fedora-based systems)
- <code>/var/log/utmp</code> - an access log that contains information regarding users that are currently logged into the system
- <code>/var/log/wtmp</code> - an access log that contains information for all users that have logged in and out of the system
- <code>/var/log/kern.log</code> - a log file containing messages from the kernel
- <code>/var/log/boot.log</code> - a log file that contains start-up messages and boot information
